# Enums and Constants

Standardized enumerations for component properties to ensure consistency across platforms.

## Button Enums

### ButtonVariant

Available button style variants:

```typescript
import { ButtonVariant } from '@toyota/core';

enum ButtonVariant {
  Primary = 'primary',
  Secondary = 'secondary',
  Ghost = 'ghost',
  Outline = 'outline'
}
```

**Usage:**

```tsx
import { ButtonVariant } from '@toyota/core';

<Button variant={ButtonVariant.Primary}>Primary</Button>
<Button variant={ButtonVariant.Secondary}>Secondary</Button>
<Button variant={ButtonVariant.Ghost}>Ghost</Button>
<Button variant={ButtonVariant.Outline}>Outline</Button>
```

**Why use enums over strings?**
- ✅ Type safety - autocomplete and error checking
- ✅ Refactoring - easy to rename values
- ✅ Documentation - clear available options
- ✅ Consistency - same values across web and native

## Size Enum

### ComponentSize

Standardized sizes for all components:

```typescript
import { ComponentSize } from '@toyota/core';

enum ComponentSize {
  Small = 'sm',
  Medium = 'md',
  Large = 'lg'
}
```

**Usage:**

```tsx
import { ComponentSize } from '@toyota/core';

<Button size={ComponentSize.Small}>Small</Button>
<Button size={ComponentSize.Medium}>Medium</Button>
<Button size={ComponentSize.Large}>Large</Button>

<Input size={ComponentSize.Medium} />
<Checkbox size={ComponentSize.Large} />
```

## Input Enums

### InputType

Available input types:

```typescript
import { InputType } from '@toyota/core';

enum InputType {
  Text = 'text',
  Email = 'email',
  Password = 'password',
  Number = 'number',
  Tel = 'tel',
  Url = 'url',
  Search = 'search'
}
```

**Usage:**

```tsx
import { InputType } from '@toyota/core';

<Input type={InputType.Email} placeholder="email@example.com" />
<Input type={InputType.Password} placeholder="Enter password" />
<Input type={InputType.Tel} placeholder="(555) 555-5555" />
```

**React Native Mapping:**

```typescript
// Web
<input type={InputType.Email} />

// React Native (maps to keyboardType)
<TextInput keyboardType="email-address" />
```

The `InputType` enum provides a common interface, but the implementation differs between platforms.

## Constants

### Default Values

Shared default values for components:

```typescript
import { 
  DEFAULT_BUTTON_VARIANT, 
  DEFAULT_SIZE 
} from '@toyota/core';
import { ButtonVariant, ComponentSize } from '@toyota/core';

// DEFAULT_BUTTON_VARIANT = ButtonVariant.Primary
// DEFAULT_SIZE = ComponentSize.Medium
```

**Usage:**

```typescript
import { DEFAULT_SIZE } from '@toyota/core';
import type { ComponentSize } from '@toyota/core';

interface ButtonProps {
  size?: ComponentSize;
}

const Button: React.FC<ButtonProps> = ({ 
  size = DEFAULT_SIZE,
  ...props 
}) => {
  // size defaults to ComponentSize.Medium
};
```

## Complete Example

### Web Component

```typescript
import React from 'react';
import { 
  ButtonVariant, 
  ComponentSize,
  DEFAULT_BUTTON_VARIANT,
  DEFAULT_SIZE
} from '@toyota/core';
import type { BaseComponentProps } from '@toyota/core';

interface ButtonProps extends BaseComponentProps {
  variant?: ButtonVariant;
}

const Button: React.FC<ButtonProps> = ({
  variant = DEFAULT_BUTTON_VARIANT,
  size = DEFAULT_SIZE,
  ...props
}) => {
  // Full type safety with enums
  const isPrimary = variant === ButtonVariant.Primary;
  const isSmall = size === ComponentSize.Small;
  
  return <button {...props} />;
};
```

### React Native Component

```typescript
import React from 'react';
import { Pressable } from 'react-native';
import { 
  ButtonVariant, 
  ComponentSize,
  DEFAULT_BUTTON_VARIANT,
  DEFAULT_SIZE
} from '@toyota/core';
import type { BaseComponentProps } from '@toyota/core';

interface ButtonProps extends BaseComponentProps {
  variant?: ButtonVariant;
}

const Button: React.FC<ButtonProps> = ({
  variant = DEFAULT_BUTTON_VARIANT,
  size = DEFAULT_SIZE,
  ...props
}) => {
  // Same enum values work on both platforms
  const isPrimary = variant === ButtonVariant.Primary;
  const isSmall = size === ComponentSize.Small;
  
  return <Pressable {...props} />;
};
```

## Benefits

### Type Safety

```typescript
// ✅ Autocomplete suggests valid options
<Button variant={ButtonVariant.} // IDE shows: Primary, Secondary, Ghost, Outline

// ❌ TypeScript error for invalid value
<Button variant="danger" /> // Error: Type '"danger"' is not assignable

// ✅ Correct usage
<Button variant={ButtonVariant.Primary} />
```

### Refactoring

If you need to rename a value:

```typescript
// Before
enum ButtonVariant {
  Primary = 'primary',
  Secondary = 'secondary',
}

// After (just update the string value)
enum ButtonVariant {
  Primary = 'main',        // Changed from 'primary'
  Secondary = 'secondary',
}

// All usages automatically updated!
```

### Cross-Platform Consistency

```typescript
// Same enum works on both platforms
import { ButtonVariant } from '@toyota/core';

// Web
const WebButton = () => (
  <button className={ButtonVariant.Primary}>Click</button>
);

// React Native  
const NativeButton = () => (
  <Pressable style={styles[ButtonVariant.Primary]}>
    <Text>Click</Text>
  </Pressable>
);
```
